# 交通費精算ツール 仕様書
バージョン: 1.0
作成日: 2026/3/4

---

## 概要

Google Maps タイムラインのエクスポートデータ（JSON）を読み込み、
会社・クライアント先への交通費を自動集計するブラウザ上で動作するツール。
集計結果は TeamSpirit への手入力補助データとして出力する。

---

## 動作環境

- 実行形式: シングルHTML（サーバー不要、ブラウザで直接開く）
- 対応ブラウザ: Chrome / Edge / Safari / Firefox（最新版）
- 外部依存: Google Fonts（Noto Sans JP, DM Mono）のみ

---

## 画面構成

### 左ペイン（サイドバー）
設定・データ読み込み操作エリア。以下のセクションで構成される。

1. データ読み込み
2. 設定（⚙️）
3. 訪問先リスト（🏢）
4. 操作ボタン
5. デバッグパネル（展開式）

### 右エリア（コンテンツ）

1. 統計バー（上部）
2. 期間ナビゲーター
3. フィルターツールバー
4. 精算一覧テーブル

## 機能仕様

### 1. タイムラインデータ読み込み

#### 入力
- ファイル形式: JSON（Google Maps タイムライン エクスポート）
- 取得方法: Android 設定 → 位置情報 → タイムライン → エクスポート
  または Google Maps アプリ → タイムライン → ⋯ → 位置情報とプライバシー設定 → 匯出時間軸資料

#### 対応フォーマット
- 新形式（端末保存形式）: semanticSegments キーを持つJSON
- 旧形式（クラウド形式）: timelineObjects キーを持つJSON
- 配列形式: レコードの配列

#### 読み込み進捗表示
ファイル選択後、以下の3ステップをリアルタイム表示する。
  ステップ1: ファイル読み込み中（MBサイズ・進捗%を表示）
  ステップ2: JSON解析中
  ステップ3: 完了（フォーマット種別・件数を表示）

#### 操作方法
- クリックしてファイル選択
- ドラッグ＆ドロップ
- 再アップロード時は自動的に前回データをリセット


### 2. 設定

#### 基本設定項目
| 項目 | 説明 | デフォルト値 |
|------|------|-------------|
| 自宅の最寄り駅（出発地） | 精算明細の出発地として表示する文字列 | 新宿 |
| 往復／片道 | 運賃の計算方式。往復の場合は登録運賃×2 | 往復 |
| 休日の扱い | 除外（表示しない）または含める（グレーアウト表示） | 除外する |

#### 設定ファイル（外部JSON）
設定の保存・再利用のために JSON ファイルとして入出力できる。

ファイル名形式: transport-expense-config_YYYY-MM-DD.json

スキーマ:
```json
{
  "version": "1.0",
  "meta": {
    "name": "設定名（任意）",
    "updatedAt": "ISO8601形式の日時"
  },
  "settings": {
    "homeStation": "出発地駅名",
    "tripMode": "round | one",
    "holidayMode": "exclude | include"
  },
  "destinations": [
    {
      "id": 数値,
      "name": "訪問先名",
      "type": "office | client",
      "fare": 片道運賃（整数・円）,
      "lat": 緯度（小数）,
      "lng": 経度（小数）,
      "radius": 判定半径（整数・メートル）,
      "color": "#HEX カラーコード"
    }
  ]
}
```

インポート: 設定セクションの「📥 インポート」ボタンからJSONファイルを選択
エクスポート: 「📤 エクスポート」ボタンで現在の設定をモーダルに出力しコピー保存

注意: Claude.ai 埋め込み環境の CSP 制限により、
      ファイルの直接ダウンロードは不可。モーダル経由のコピー＆ペースト保存を使用する。


### 3. 訪問先リスト

各訪問先は以下の属性を持つ。

| 属性 | 説明 |
|------|------|
| 名前 | 訪問先の表示名 |
| 種別 | office（通勤）または client（訪問） |
| 片道運賃 | 円単位の整数 |
| 緯度・経度 | Google Maps で右クリック→座標コピーで取得 |
| 判定半径 | タイムラインの座標がこの半径内に入れば「訪問あり」と判定（メートル） |
| カラー | 一覧表示時の識別色（自動割り当て） |

#### マッチングアルゴリズム
ハーバーサイン公式（Haversine formula）を使用して2点間の距離（メートル）を計算し、
登録された全訪問先との距離を比較。判定半径内に入る最も近い訪問先にマッチさせる。
複数の訪問先が半径内に入る場合は最も近い1件を採用する。

#### デフォルト設定
初期状態では「秋葉原（会社）」が1件登録されている（仮の座標・運賃）。
実際の場所に合わせて設定ファイルで上書きすること。

#### 重複除去
同じ日付・同じ訪問先の組み合わせは1件に集約する。


### 4. 期間指定

| 操作 | 説明 |
|------|------|
| ◀ 前月 / 次月 ▶ | 月単位で前後に移動 |
| 今月 | 現在の年月を選択（起動時のデフォルト） |
| 先月 | 前月を選択（通常の精算対象月） |
| 日付入力欄（〜） | 任意の開始日・終了日を直接指定（範囲モード） |

月モードと範囲モードは自動的に切り替わる。
統計バーの集計値も選択中の期間に連動する。


### 5. 休日判定

#### 非営業日の定義
- 土曜日（曜日番号: 6）
- 日曜日（曜日番号: 0）
- 日本の祝日（ハードコードされたリスト）

#### 組み込み祝日データの範囲
2024年〜2026年3月までの日本の祝日を内包。
振替休日も含む。

#### 休日の扱い設定による挙動
- 除外: 休日の精算データは一覧に表示されず、集計にも含まれない
- 含める: 休日データはグレーアウト表示され、チェックボックスで個別に精算対象から外せる


### 6. 精算一覧テーブル

#### 表示項目
チェック / 日付 / 曜日 / 出発地 / → / 訪問先 / 種別タグ / 運賃

#### 種別タグ
- 通勤（青）: office 種別の訪問先
- 訪問（紫）: client 種別の訪問先
- 休日（赤）: 土日祝日
- 往復（緑）: 往復設定時
- 片道（黄）: 片道設定時

#### チェックボックス
各行のチェックボックスで精算対象を個別に除外できる。
統計バーの合計金額に即座に反映される。

#### 月グループヘッダー
範囲モード使用時など複数月にまたがる場合、月ごとにグループヘッダーと月別小計を表示する。

#### フィルター
- すべて: 全訪問先を表示
- 通勤: office 種別のみ表示
- 訪問: client 種別のみ表示
- 休日を表示/非表示: 休日データの表示切り替え


### 7. 統計バー

選択中の期間・チェック状態に基づいてリアルタイム集計。

| 項目 | 説明 |
|------|------|
| 精算対象日数 | チェック済みの件数 |
| 合計交通費 | チェック済みの運賃合計（円） |
| 会社への通勤 | office 種別のチェック済み件数 |
| クライアント訪問 | client 種別のチェック済み件数 |


### 8. 出力機能

#### TeamSpirit用コピー（📋）
チェック済みの精算データを月別テキスト形式でクリップボードにコピーする。
TeamSpirit への手入力時の参照データとして使用する。

出力形式:
  【TeamSpirit 交通費精算データ】
  出力日時: YYYY/MM/DD HH:MM:SS
  ■ YYYY年M月 (小計: ¥X,XXX)
  YYYY-MM-DD（曜）　出発地 → 訪問先　往復/片道　¥XXX
  ─────────────────
  合計: ¥X,XXX

#### CSV出力（⬇）
チェック済みの精算データをCSV形式でモーダルに出力しコピー保存する。
BOM付きUTF-8。Excelでの文字化け防止済み。

出力列: 日付 / 曜日 / 出発地 / 訪問先 / 種別 / 往復片道 / 運賃(円)


### 9. デバッグ機能（🔎 データ内の場所名を確認する）

タイムラインJSONから抽出した訪問地点の座標一覧を表示する。
各行をクリックすると緯度・経度をクリップボードにコピーできる。
訪問先登録時の座標取得に活用する。

表示情報:
- 場所名（Google Maps が自動付与）
- 緯度・経度（小数5桁）
- 初回訪問日
- 統計（visit総数 / 座標あり件数 / ユニーク地点数）
- 最初のvisitデータのRAW JSON（展開式）

---

## データフロー

1. タイムラインJSON読み込み → semanticSegments を解析
2. 各visitセグメントから日付・緯度経度を抽出
3. ハーバーサイン距離計算で訪問先とマッチング
4. 同日・同訪問先の重複を除去
5. 土日祝判定を付与
6. 期間フィルター・種別フィルターで絞り込み
7. テーブル表示 / 統計集計

---

## 既知の制約

- Claude.ai 埋め込み環境では blob: URL / data: URI によるファイルダウンロードが
  CSP（Content Security Policy）でブロックされるため、
  エクスポートはモーダル経由のコピー＆ペースト保存となる。
- 祝日データは 2024〜2026年3月分をハードコード。それ以降の祝日は手動対応が必要。
- TeamSpirit への自動入力APIは未対応（手入力補助ツールとして運用）。
- タイムラインの座標精度は Google Maps の記録精度に依存する。
  精度が低い場合は判定半径（radius）を広げて対応する。
- GitHub連携のPATはSessionStorageに一時保存。ページを閉じると消去される。
